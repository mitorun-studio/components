Раскрывающийся контент с JS Bootstrap Collapse

Этот скрипт и соответствующие стили позволяют делать раскрывающийся контент, который появляется и изчезает при нажатии на кнопку(-и). Например бургер-меню, попапы и т.п.

Для управляющей кнопки нужно задать атрибуты data-bs-toggle="collapse" и data-bs-target="#collapseElement" (это id или class раскрывающегося элемента(-ов)!). А для самого раскрывающегося элемента нужно добавить класс .collapse для его скрытия и идентификатор или класс по которому с ним будет связана кнопка. Для создания группы спойлеров дополнительно можно добавить атрибуты data-bs-parent="#selector" (подробнее в компоненте спойлер Bootstrap Accordion

Класс .collapse скрывает элемент с display: none;, класс .collapsing добавляется скриптом во время анимации(!), класс .collapse.show показывает скрытый элемент.

Свернутый контент имеет высоту 0px, а при открытии ему добавляется высота с применением анимации.

Анимация раскрытия может стать дерганной если элемент (и его потомки?) содержит маржины и паддинги, так как они не анимируются и появляются мгновенно!

Добавление для элемента класса collapse show сделает контент видимым при загрузке страницы, при нажатии на кнопку его также можно закрыть. Но тогда нужно для управляющей кнопки поставить атрибут aria-expanded="true"!

По умолчанию стили написаны для вертикального раскрытия контента. Чтобы сделать раскрытие по горизонтали надо добавить класс .collapse-horizontal и установить ширину для дочернего элемента! Такую горизонтальную раскрывашку можно спозиционировать и сделать раскрывающееся боковое меню.

Можно создавать мульти-всплывашки и мультикнопки для них! Пример в документации бутстрапа.

Раскрывающиеся боковые панели с JS Bootstrap Offcanvas

Bootstrap Offcanvas очень похож визуально и функционально на Bootstrap Modal, но это отдельные плагины (хотя и с общими стилями).

По умолчанию при открытии Offcanvas появляется оверлей и отключается прокрутка body. Это можно отключить с помощью атрибутов data-bs-scroll="true" data-bs-backdrop="false" для .offcanvas . Можно убрать возможность закрывать Offcanvas с помощью Esc атрибутом data-bs-keyboard="false"

Нельзя использовать margin (, padding?) и/или translate для элемента .offcanvas из-за особенностей обработки анимации.

Доступность: скрипт автоматически добавляет атрибут role="dialog".

Выпадающие списки с JS Bootstrap Dropdown

Dropdown основаны на сторонней библиотеке Popper, которая обеспечивает динамическое позиционирование выпадашки. Библиотека Popper содержится (только) в bootstrap.bundle.min.js!

Popper вроде как не используется в выпадашках в навигационном меню Navbar! Поэтому если использовать дропдаун только там, то не нужно подключать Popper JS (полный файл bootstrap.bundle.min.js).

Доступность: работает управление с клавиатуры (перемещение стрелками и выход с Esc). Для выпадашек нужно самостоятельно добавлять подходящие ариа-роли (например role="menu").

Управлять отступом между раскрытым списком и кнопкой можно с помощью атрибутов data-bs-offset="10,20" и data-bs-reference="parent" для кнопки раскрытия.

Методы закрытия раскрытого списка настраиваются атрибутами для кнопки: data-bs-auto-close="true" – значение по умолчанию (закрывается от клика по кнопке, по списку, вне всего дропдауна и Esc), data-bs-auto-close="inside" – не закрывается кликом вне дропдауна, data-bs-auto-close="outside" – не закрывается от клика внутри списка, data-bs-auto-close="false" – закрывается только от клика по кнопке.

Есть и другие настройки с помощью дата-атрибутов (или в JS) в документации. По умолчанию включено динамическое позиционирование Popper data-bs-display="dynamic", его можно отключить с помощью значения "static".

Всплывающие сообщения с JS Bootstrap Toast

Документация. Тосты — это легкие уведомления, имитирующие push-уведомления, популярные в мобильных и настольных операционных системах.

Popover по умолчанию отключены из-за соображений производительности, их нужно активировать вручную! Вот один из способов инициализировать ВСЕ поповеры на странице по их дата-атрибуту: var toastTrigger = document.getElementById('liveToastBtn') var toastLiveExample = document.getElementById('liveToast') if (toastTrigger) { toastTrigger.addEventListener('click', function () { var toast = new bootstrap.Toast(toastLiveExample) toast.show() }) }

Как работает: в скрытом состоянии скрипт добавляет к .toast классы .fade.hide . При открытии на миг добавляется класс .showing, а после этой анимации класс .show . Повторный клик по кноке закроет тост?

Настройки: Тосты по умолчанию будут автоматически скрываться через (5с?), изменить время скрытия можно атрибутом data-bs-delay="10000"? для отключения автоскрытия прописать autohide: false. Выключить анимацию с data-bs-animation="false".
