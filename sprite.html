SVG спрайты

Какие есть требования к спрайтам:

  • Несложно создавать вручную. SVG-иконки для спрайта должны иметь те же точки, что и отдельные иконки. Один файл спрайта для всех иконок и фигур сайта (в идеале чтобы отдельные иконки могли быть сделаны с разными размерами вьюбокса, например 9x9px или 96x96px).
  • Возможность перекрашивать в CSS (менять цвет заливки и обводки).
  • Легкое использование с минимумом кода. Чтобы для вставки каждой иконки спрайта не нужно было прописывать стопку свойств ее положения, размера и цвета.
  • Можно использовать и в HTML (контентный img?) и в CSS фоном. И SVG и растровые картинки.

Спрайт №0 (SVG-stack, SVG-стеки)

Это выглядит самым универсальным способом. Его особенности:

  • Легко создавать спрайт вручную - просто вставлять внутрь спрайта SVG-файлы отдельных иконок. Также легко эти отдельные иконки вынуть из спрайта и отредактировать. Минус только в том, что просто так не видно что лежит в файле спрайта, пока его не подключишь.
  • Спрайт можно подключать для фона (CSS background), для контента (HTML img), для svg use (HTML svg), а также в HTML embed и HTML iframe.
  • Менять картинку при ховере можно при подключении как фон. Для остальных способов если и можно, то сложно (пока не знаю как). Например для HTML svg-use можно при ховере сделать цвет прозрачным и добавить фоном другую картинку!
  • Менять цвет заливки и обводки элементов SVG картинки можно вроде только в варианте HTML svg-use.

Картинка в HTML в svg

Можно менять цвета заливки и обводки (всех отдельных фигур в SVG). Можно чтобы цвета SVG зависели от цвета элемента currentColor.
Минусы: не работает SVG SMIL анимация, мусорится разметка в HTML, надо скрывать элемент от скринридеров, повторять код для каждого элемента (плохо для повторяющихся элементов).

Контентная картинка в HTML в img

Можно менять картинку (например в темной теме, при адаптиве или при ховере) в самом HTML в теге picture!

Картинка в CSS в background-image

Можно легко менять картинку сменой ссылки. Гибко настраивается размер, расположение, повторение картинки. Отлично подходит для повторяющихся иконок типа маркера галочки у списка.

Картинка в CSS в mask-image

Спрайт №1 (обычный со сменой позиции)

Это (простейший?) спрайт, он создается объединением по горизонтали квадратных картинок (SVG-иконок). Далее его можно использовать для контентных картинок или фоновых, меняя положение вывода картинки.

Этот способ рабочий, можно применять когда нужно только выводить разные картинки, особенно подходит для растра. Но нельзя менять цвета SVG внутри такого спрайта.

Спрайты №2 (SVG-спрайты c symbol)

В этом способе спрайт создается более сложным способом с тегами <symbol>. Спрайт можно использовать только в HTML, в теге <svg>. Его можно перекрашивать (указывая fill="currentcolor").

Спрайт №3 (SVG-фрагменты, с view)

Это симбиоз способов №1 и №2. Спрайт создается сложным способом с тегами <view> и атрибутами transform="translate()". Его можно использовать в img или фоне, очень легко - просто прописывая id (этим он лучше способа №1).

Минусы: Нельзя перекрашивать. Сложно создавать спрайт. (Нельзя менять img-картинку при наведении?)

Спрайт №3-2 (SVG-фрагменты, без view)

Этот способ аналогичен №3, но тут нет тега <view> и ID, а вместо ID сразу пишется местоположение вьюбокса в ссылке!

Имеет все те же недостатки и плюсы, как и в способе №3.
