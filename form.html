Формы связи

Правила компонента:

  • Форма без классов, только с тегами, не должна иметь своих стилей оформления. Или ей можно задать базовые отступы, чтобы нормально смотрелись дефолтные элементы.
  • Для кастомных стилей назначаются "базовые" классы .form, .form__title, .form__label, .form__input, .form__item-point . Различать типы инпутов можно по атрибуту type[].
  • Другие темы оформления форм включаются простым добавлением класса для тега <form>, а в стилях этот класс подписывается перед каждым селектором. При добавлении компонента с темой на сайт можно объединить базовые классы формы с классами темы.
  • Для добавления стилей встроенной валидации нужно после текстового инпута поставить тег <span class="form__validation" aria-hidden="true"></span> и добавить стили для валидации инпута и значка form__validation, и у инпута должен быть плейсхолдер (хотя бы пробел туда поставить). Эти стили валидации настроены показывать только невалидные (invalid) поля для обязательных (required) инпутов, причем они работают только если пользователь что-то ввел, то есть заменил плейсхолдер на свой текст. Чтобы показывалась валидация не всех обязательных инпутов (например в тех инпутах где нет плейсхолдера!) - можно для стилей валидации сделать отдельный класс .validity .

Что нужно сделать:
- Для всплывающих сообщений об успешной/неуспешной отправке сообщений нужно сделать плавную анимацию появления. Хотя бы как у элемента details, а лучше как это сделано в бутстрапе с задержкой появления(?).
- В будущем можно переделать цветовые стили инпутов с accent-color.

Форма №2: лейблы на инпутах и отъезжают вверх при фокусе (нестандартная разметка HTML)

Особенность этой формы: label должен идти после input, а в плейсхолдере должен стоять пробел чтобы подпись анимировалась и отъезжала наверх, это из-за особенности хитрых селекторов!

Для стилизации этой формы надо взять обычные стили базовой формы и дополнить их уникальными стилями для .form--2. Ещё нюанс в том, что для form__item убирается display: grid;, поэтому для валидации уже не стоит использовать отдельный элемент .form__validation для иконки (!), который в базовой форме располагается с помощью гридов.
